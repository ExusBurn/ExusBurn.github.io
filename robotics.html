System Overview

Modern robotic systems face a fundamental challenge: how do you build a robot that can navigate an unknown environment, identify objects within it, and physically interact with them — all in real time, with the reliability required for practical deployment?

This project addresses that challenge through a tightly integrated system that combines three core capabilities: autonomous navigation via Visual SLAM, real-time object detection via YOLOv3, and a 3D grasping pipeline that computes object position and orientation from camera data and executes a physically accurate grasp with a 7-DOF arm.

The system is designed to operate in indoor environments without prior maps or object templates. It builds its understanding of the environment from scratch using only onboard cameras, and its object detection generalises to novel instances within the target class.

Visual SLAM: Autonomous Navigation

Simultaneous Localisation and Mapping (SLAM) is the problem of building a map of an unknown environment while simultaneously tracking the agent's location within it — a chicken-and-egg problem that requires solving both tasks jointly.

Visual SLAM solves this using only camera images, without GPS or external positioning systems. Our implementation tracks visual features (corners, edges) across consecutive frames to estimate camera motion, builds a sparse 3D map of the environment from triangulated feature correspondences, and uses loop closure detection to correct accumulated drift when the robot revisits a previously seen location.

The SLAM module outputs a real-time estimate of the robot's 6-DOF pose (position + orientation) in the map frame, which feeds into the navigation planner. The robot plans paths in the SLAM map, avoiding obstacles detected as occupied regions, and drives autonomously to target locations.

Why Visual SLAM? Traditional SLAM uses LiDAR, which is expensive and bulky. Visual SLAM uses standard cameras — cheaper, lighter, and richer in semantic information. The tradeoff is computational cost and sensitivity to lighting changes, both of which are managed in our implementation.

Object Detection: YOLOv3

Object detection is handled by YOLOv3 (You Only Look Once, version 3), a single-stage real-time detector that predicts bounding boxes and class probabilities directly from image pixels in one forward pass.

YOLO's architecture divides the input image into a grid and predicts bounding boxes and class probabilities at each grid cell. At YOLOv3's scale — processing at multiple resolutions — it achieves strong detection of objects at different sizes. Critically, it runs in real time: fast enough to track objects as the robot moves and as the target may shift position.

The detection system is used for two distinct purposes:

  • Initial detection: Identifying the target object in the scene and localising its 2D image position as the starting point for the grasping pipeline.
  • Tracking: Monitoring the object across frames to detect any sudden positional changes — if the target moves between detection and grasp, the system updates its estimate and adjusts the arm trajectory accordingly.

The Grasping Pipeline

Knowing where an object is in a 2D image is not enough to grasp it — the robot needs the object's 3D position and orientation in the robot's coordinate frame. The grasping pipeline solves this in three steps.

Step 1: Contour-Based 3D Localisation

Once YOLOv3 provides a bounding box, a contour detection algorithm extracts the precise boundary of the object within the box. Fitting an ellipse or polygon to this contour gives a sub-pixel estimate of the object's centroid and principal axes in image space.

Depth information (from a depth camera or stereo rig) is used to back-project the 2D centroid into 3D space. The result is the object's position in the camera frame, which is then transformed to the robot base frame using the camera-to-robot extrinsic calibration.

Step 2: Orientation Estimation

Grasping an object correctly requires knowing not just where it is but how it is oriented — the gripper must approach from the right angle to achieve a stable grasp. Our system uses a multi-angle camera setup: by viewing the object from two or more positions, we estimate the object's 3D orientation through geometric reasoning.

For elongated objects (bottles, tools), the principal axis of the contour gives the dominant orientation directly. For symmetric objects, the system uses depth gradients to distinguish the top surface and plan a grasp perpendicular to it.

Step 3: Arm Control & Execution

With the 3D target pose (position + orientation) determined, the arm's inverse kinematics solver computes the joint angles required to place the end-effector at the grasp point with the correct approach direction. The 7-DOF arm has enough degrees of freedom to reach the target in a wide range of configurations, allowing the controller to choose a collision-free path.

The grasp is executed in two phases: a pre-grasp approach from a safe standoff distance, followed by a final closing motion. The gripper closes on the object and lift is confirmed by a force/torque threshold.

Human-Following

The system also implements a human-following mode: the robot detects a person via YOLO, tracks their position across frames, and drives the rover base to maintain a fixed distance. Payloads are transported to the person, placed in proximity, and the robot returns to standby. This demonstrates the integration of SLAM navigation, real-time tracking, and mobile manipulation in a single coordinated system.

Experimental Results

The system was evaluated across a range of objects — varying shape, size, weight, and surface texture — in an indoor lab environment. The robot was given a target object, tasked with navigating to it, detecting it, and executing a grasp.

Grasping success rate: 94% across experimental trials. Failures were concentrated in cases of specular reflection on glossy surfaces (confusing the depth estimate) and partial occlusion of the target by clutter.

Navigation via Visual SLAM succeeded in all test environments, including environments not seen during any calibration phase. Loop closure detection correctly identified revisited locations and corrected pose drift in all cases.
